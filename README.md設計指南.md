1. 專案封面區

樣式:
- 使用置中排版。
- 最上方可放專案 Logo 或主要截圖縮圖。
- 圖片用 HTML 放置，方便控制置中與寬度。
- 專案名稱用置中的 `<h1>`。
- 副標用一段置中的粗體說明文字。
- 技術棧用 badge 橫向排列。

圖片寫法:
<p align="center">
  <img src="./public/logo.png" width="120" alt="Project Logo" />
</p>

標題寫法:
<h1 align="center">專案名稱</h1>

副標寫法:
<p align="center">
  <strong>一句話說明這個專案是什麼</strong>
</p>

badge 寫法:
<p align="center">
  <img src="https://img.shields.io/badge/Vue-3-42B883?style=flat-square&logo=vuedotjs" />
  <img src="https://img.shields.io/badge/TypeScript-5-3178C6?style=flat-square&logo=typescript" />
</p>
```

2. 簡介

樣式:
- 使用二級標題 `## 📖 簡介`。
- 下面是一段說明文字。
- 第一段說明「這是什麼專案」。
- 第二段說明「解決什麼痛點」。
- 重要專案名稱可以用粗體。

文字形式:
## 📖 簡介

**專案名稱** 是一個為 XXX 打造的 XXX 系統。

這個專案主要解決 XXX 問題，透過 XXX、XXX、XXX 等功能，讓使用者可以更有效地完成 XXX。
```

```md
3. 為什麼做這個專案

樣式:
- 使用三級標題。
- 使用 bullet list。
- 每一點建議格式為：emoji + 粗體短標 + 一句說明。
- 適合放 3～6 點，不要太長。

寫法:
### 為什麼做這個專案？

- 🔍 **缺乏搜尋功能**：說明原本痛點與你的解法。
- 📦 **缺少獨立頁面**：說明使用者為何需要這個功能。
- 🚀 **建立成本過高**：說明你的專案如何降低門檻。
- ⭐ **缺乏互動機制**：說明評論、收藏、分享等互動價值。
```

```md
4. 功能特色

樣式:
- 使用二級標題 `## ✨ 功能特色`。
- 如果是頁面規劃，使用表格。
- 如果是功能說明，使用小標題 + bullet list。
- 每個功能區塊建議使用三級標題。

頁面規劃表格:
### 🧭 頁面規劃

| 頁面 | 路由 | 說明 |
|---|---|---|
| 首頁 | `/` | 搜尋框 + 熱門內容 |
| 搜尋結果 | `/search` | 搜尋、排序、篩選 |
| 詳情頁 | `/item/[id]` | 詳細資訊與操作按鈕 |

功能列表:
### 🔍 搜尋與排序

- **搜尋範圍**：商品名稱、分類、描述。
- **排序選項**：熱門程度、更新時間、價格。
- **篩選條件**：價格區間、分類、狀態。
```

```md
5. 技術架構

樣式:
- 使用表格最清楚。
- 欄位建議為：層級 / 技術 / 說明。
- 不要只列技術名稱，要說明每個技術負責什麼。

寫法:
## 🏗️ 技術架構

| 層級 | 技術 | 說明 |
|---|---|---|
| 前端框架 | Vue 3 / Nuxt | 建立主要前端架構 |
| UI | Tailwind CSS | RWD 與客製化樣式 |
| 狀態管理 | Pinia | 管理全站狀態 |
| 後端 | Node.js / API Routes | 處理資料與商業邏輯 |
| 資料庫 | PostgreSQL | 資料儲存與查詢 |
| 部署 | Vercel / Cloudflare | 上線與 CDN |
```

````md
6. 使用指南

樣式:
- 使用二級標題。
- 依使用流程拆成多個編號小標題。
- 如果是一般使用者操作，用 numbered list。
- 如果是開發者安裝，用 code block。

使用者操作:
## 📚 使用指南

### 1. 匯入資料

1. 前往 `/import` 頁面。
2. 貼上資料來源網址。
3. 預覽匯入內容。
4. 確認後建立資料。

### 2. 搜尋內容

1. 在搜尋欄輸入關鍵字。
2. 選擇排序方式。
3. 使用篩選條件縮小結果。

開發者操作:
### 安裝專案

```bash
npm install
````

### 啟動開發環境

```bash
npm run dev
```

7. API / 系統流程

樣式:
- API 適合用表格。
- 系統流程適合用文字流程圖。
- 如果 API 有防護機制，可以多一欄「防護」。

API 表格:
## 🔌 API

| Endpoint | 說明 | 防護 |
|---|---|---|
| `GET /api/items` | 取得資料列表 | - |
| `POST /api/items` | 建立資料 | Auth / Rate Limit |
| `POST /api/upload` | 上傳檔案 | Auth / File validation |

系統流程:
### 資料處理流程

```txt
使用者提交資料
→ API 驗證
→ 寫入資料庫
→ 回傳結果
→ 前端更新畫面
````

````

8. 專案成果 / 技術亮點

樣式:
- 使用二級標題。
- 底下拆成 2～3 個小段落。
- 每段使用三級標題 + bullet list。
- 適合寫「技術亮點」、「產品特色」、「開發成果」。

寫法:
## ✅ 專案成果

### 技術亮點

- 使用 XXX 架構解決 XXX 問題。
- 導入 XXX 提升效能與穩定性。
- 透過 XXX 完成自動化流程。

### 產品特色

- 支援一鍵操作。
- 提供完整搜尋與篩選。
- 具備使用者互動機制。

### 開發成果

- 完成完整產品流程。
- 建立可維護的前端架構。
- 完成部署與線上展示。
````

9. 未來規劃

樣式:
- 使用 checkbox list。
- 已完成可用 `[x]`，未完成用 `[ ]`。
- 適合展示你對專案後續發展有規劃。

寫法:
## 🧩 未來規劃

- [x] 完成核心功能開發
- [x] 完成 RWD 響應式設計
- [ ] 新增會員系統
- [ ] 新增後台管理
- [ ] 新增資料視覺化儀表板
- [ ] 優化 SEO 與效能
```
